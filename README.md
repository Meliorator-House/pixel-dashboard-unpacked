# Jedi Focus Dashboard

Готовый MVP сайта-приложения в pixel-art стиле: задачи дня, календарь, проекты, финансы, цели, привычки, 3D-статистика дня и пиксельный дельфин-помощник.

## Что внутри

```txt
jedi-focus-dashboard/
├─ src/
│  ├─ App.tsx          # весь интерфейс и логика MVP
│  ├─ main.tsx
│  └─ styles.css       # pixel-art/glassmorphism дизайн
├─ bot/
│  ├─ bot.py           # Telegram-бот на Python
│  ├─ server.py        # FastAPI-сервер для событий с сайта
│  ├─ data_store.py    # JSON-хранилище
│  ├─ requirements.txt
│  ├─ .env.example
│  └─ README.md
├─ package.json
├─ vite.config.ts
└─ .env.example
```

## Заливка сайта на Vercel

1. Создай репозиторий на GitHub.
2. Загрузи туда файлы проекта.
3. В Vercel нажми **New Project**.
4. Выбери репозиторий.
5. Framework: **Vite**.
6. Build command: `npm run build`.
7. Output directory: `dist`.

## Telegram-бот

Python-бот лежит в папке `/bot`.

Сайт можно держать на Vercel, а бота лучше отдельно:

- Railway
- Render
- Fly.io
- VPS
- PythonAnywhere

Почему отдельно: боту нужно работать постоянно, чтобы ловить дедлайны и отправлять уведомления. Vercel больше подходит для сайта, а не для постоянного Python polling-бота.

## Связка сайта и бота

В MVP сайт хранит данные в `localStorage`.

Чтобы сайт отправлял события в Telegram через бота:

1. Разверни `/bot/server.py` на Render/Railway/VPS.
2. Получи публичный URL, например:

```txt
https://jedi-focus-bot.onrender.com
```

3. В настройках сайта вставь этот URL в поле **Bot API URL**.

Также можно указать переменную окружения:

```env
VITE_BOT_API_URL=https://jedi-focus-bot.onrender.com
```

## Важный момент

Полная синхронизация сайта и бота через один общий баланс/задачи на 100% требует базы данных: Supabase, Firebase или PostgreSQL.

В этом MVP:
- сайт хранит данные в браузере;
- бот может получать события с сайта через FastAPI;
- бот хранит свою копию данных в JSON.

Это нормальная основа, которую потом можно расширить до полноценной AutoCRM.
